# 珂溪头疗 AI 智能化管理系统 - 产品需求文档 (PRD)

## 1. 产品概述
**1.1 产品背景**
传统头疗/美业门店在运营过程中，面临预约排班繁琐、知识库难以沉淀、财务数据分散、缺乏数据洞察等痛点。本项目旨在为“珂溪头疗”打造一款深度融合 AI 能力的一站式管理平台。

**1.2 产品定位**
一个以“自然语言交互（Chat）”为核心入口，辅以“多维可视化看板”的现代化门店管理 SaaS 系统。通过引入 AI 智能体（Agents），降低员工学习成本，实现门店运营、财务、排班、培训的自动化与智能化。

---

## 2. 整体产品架构
系统采用 **“左侧隐藏/折叠菜单 + 右侧核心工作区”** 的经典布局，支持明暗双色主题（基于参考 UI 的奶油色与深灰色系）。
核心模块包括：
1. **AI 工作台 (首页)**：全局对话入口，多智能体切换。
2. **经营看板**：多维度数据图表与统计。
3. **预约与排班**：客户预约、员工排班、请假管理。
4. **财务数据**：收支管理、财务报表、AI 财务洞察。
5. **知识库与文档**：企业沉淀、对话式录入、文档查阅。
6. **系统设置**：API Key 配置、权限管理。

---

## 3. 核心功能需求详述

### 3.1 首页：AI 聊天工作台与智能体矩阵 (需求 #1)
首页是用户使用频率最高的模块，采用类似 Gemini 的聊天流设计。

*   **功能描述**：
    *   **核心对话框**：支持文本输入、语音输入。支持一键发起对话。
    *   **智能体选择器 (Agent Selector)**：顶部横向滑动的智能体切换栏，不同智能体连接不同的私有数据和权限。
*   **【深度分析】推荐的智能体矩阵**：
    1.  **默认智能体 (全局路由)**：能听懂所有指令，自动将任务分发给具体的专业智能体。
    2.  **排班预约官**：
        *   *用途*：处理客户预约、技师排班查询。
        *   *场景*：“帮我看看徐汇店今天下午3点还有哪位技师有空？”或“帮我把小李明天的班调成早班”。
    3.  **财务分析师**：
        *   *用途*：财务数据查询与分析，严格受权限控制。
        *   *场景*：“分析一下上个月各门店的净利润对比”，或者“为什么这个月耗材成本超标了？”
    4.  **头疗业务专家 (培训/知识库)**：
        *   *用途*：解答头皮护理、精油配比、理疗手法等专业问题。
        *   *场景*：“客人头皮容易出油长痘，推荐使用什么项目和精油？”
    5.  **客服助手**：
        *   *用途*：生成话术，辅助回复客人微信或评价。

### 3.2 知识库与文档中心 (需求 #2, #8)
改变传统手动录入的繁琐，利用 AI 实现“对话即沉淀”。

*   **功能描述**：
    *   **对话式导入 (核心亮点)**：员工可以将日常与客户的聊天记录、解决问题的过程直接复制发给 AI（或通过专门的“对话导入”入口）。AI 会自动提取核心问题（FAQ）、治疗技术点或客户偏好，并归类整理到知识库中。
    *   **分类管理**：支持将知识库划分为“治疗技术”、“常见问题 (FAQ)”、“产品手册”、“规章制度”等。
    *   **文档查看器**：支持在线预览公司各类文档（PDF、图片、Word 等）。
    *   **AI 智能合成审核**：AI 自动从近期对话中提取出的知识点，进入“待审核”状态，管理员确认后即刻入库。

### 3.3 财务管理模块 (需求 #3)
基于常规门店财务需求，深度融合 AI 辅助分析。

*   **常规财务功能**：
    *   **收支明细 (近期交易记录)**：记录每一笔订单流水、耗材采购、会员充值。
    *   **收入管理 & 支出分析**：记录人工成本、房租水电、耗材采购、市场营销等。
    *   **财务报表**：月度/季度总收入、运营成本、净利润核算。
*   **AI 融合功能 (隐藏菜单中的亮点)**：
    *   **AI 智能洞察面板**：自动生成财务诊断报告（例如：“本月利润率提升了5%，主要归功于头皮深度护理项目的销售增长”）。
    *   **降本增效建议**：AI 扫描支出结构，自动提示异常支出（如：“本月耗材消耗相比客单量偏高，建议排查”）。
    *   **营收预测**：基于历史数据模型，预测下月营收，并在图表上展示。

### 3.4 经营数据可视化看板 (需求 #4)
直观的图表展示，让管理者随时掌握全局。

*   **功能描述**：
    *   **门店切换**：支持“全店概览”与“单店数据（如静安店、徐汇店）”切换对比。
    *   **核心指标卡片**：今日预约数、今日营收、接待客人数及环比增长率。
    *   **可视化图表**：
        *   营收统计趋势线（支持7天/30天切换）。
        *   服务项目分布饼图/环形图（掌握哪些项目最受欢迎）。
        *   客流预测分析柱状图（对比实际客流与预测客流）。
    *   **各门店实时业绩榜单**：展示单量、转化率、客户评价。

### 3.5 预约排班与人事管理 (需求 #5, #7)
将静态的排班表变为可对话的动态数据。

*   **功能描述**：
    *   **预约列表视图**：展示各时间段的预约详情（客户信息、服务项目、指定技师、状态）。
    *   **数据导入**：支持一键导入每个门店、每个时间段的预约记录（Excel/系统对接）。
    *   **技师排班表**：周视图呈现员工的早班、中班、晚班及休息状态。
    *   **请假管理**：员工请假申请列表，支持一键审批。
    *   **AI 深度介入场景**：
        *   *动态查询*：店长直接在首页问：“这周末哪几个理疗师休假？”
        *   *智能排班诊断*：AI 分析下周预约趋势，主动提示：“系统检测到下周三下午预约量预计增长 30%，建议提前调整排班”。

### 3.6 系统设置 (需求 #6)
保证系统的灵活性与可扩展性。

*   **功能描述**：
    *   **模型配置**：提供专门的输入框，供管理员输入并保存 **智谱 API KEY**（或切换其他模型源）。
    *   **权限管理**：设置不同角色的访问权限（店长、前台、技师、财务）。
    *   **门店配置**：新增或修改门店基础信息。

## 4. UI/UX 风格与交互规范 (基于提供的代码与设计参考)

*   **视觉基调**：
    *   **品牌色**：采用高级暖金色 `Primary: #b6860c`，传递美业、头疗的放松与高端感。
    *   **主题支持**：必须支持 Light（明亮，背景 `#f8f7f6`）和 Dark（暗黑，背景 `#221d10`）双模式，满足不同光线环境下的使用需求。
    *   **版式**：采用大圆角 (`rounded-xl`, `rounded-2xl`)、毛玻璃效果 (`backdrop-blur-md`) 以及柔和的阴影，界面需留白充分，避免视觉拥挤。
*   **核心交互**：
    *   **隐藏式侧边栏**：左侧菜单在小屏幕上可折叠收起为图标模式，主工作区保持沉浸式体验。
    *   **无缝滚动**：利用 `custom-scrollbar` 隐藏原生滚动条，提升页面精致度。
    *   **悬浮输入框**：聊天界面的输入框采用底部悬浮设计，带有呼吸光效（`group-focus-within`），强调 AI 交互的核心地位。

## 5. 项目推进建议 (Next Steps)
1.  **后端架构搭建**：由于涉及多个业务模块与 AI 的结合，建议后端采用微服务或模块化设计，并将业务数据（如排班表、财务表）进行向量化（Vector DB）或结构化处理，以便智谱 API 能够精准调用。
2.  **API 联调测试**：优先走通“智谱 API”的接入，测试“意图识别（将用户对话分发给财务或排班）”的准确率。
3.  **前端组件化开发**：您的 HTML 原型已经非常完善，可以直接转化为 Vue/React/Svelte 组件，重点处理数据状态管理（如切换门店时，全局图表数据更新）。
